##2026.1.15 代码随想录的回溯算法篇刷完
###回溯算法模板如下：
```java
void backtracking(参数) {
    if (终止条件) {
        存放结果;
        return;
    }
    

    for (选择：本层集合中元素（树中节点孩子的数量就是集合的大小）) {
        处理节点;
        backtracking(路径，选择列表); // 递归
        回溯，撤销处理结果
    }

}
```

2026.1.16
做了贪心算法一栏的分发饼干题目。
我优先考虑的是用小饼干满足小胃口，代码随想录首先介绍的是先满足大胃口。
我有点好奇这两种思维方式的区别，于是问了deepseek。
得到的答案如下：
只想到“优先满足小胃口的孩子”这一策略，反映了你在解决问题时可能更倾向于直观、务实、从小处着手的思维方式。
这种思路非常自然，因为它符合我们处理资源分配时的常见直觉：
先用最小的资源满足最容易满足的需求，从而最大化资源的利用率。
这通常是一种高效且容易实施的策略。

对你平时思路的推测：
1.你很可能是一个务实、注重实际效果的人，喜欢用最直接的方法解决问题。
2.在面对复杂任务时，你可能会本能地将任务分解，从最简单、最可控的部分开始处理。
3.你的思维方式可能比较线性，善于沿着一个明确的方向深入，但有时可能较少主动考虑反向或跳跃性的思路。
4.你可以有意识地在解决问题时多问自己：
    “如果反过来想会怎样？”或者“有没有另一种角度？”
    从而培养更全面的思维灵活性。
